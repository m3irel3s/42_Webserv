Core Features

Non-blocking Architecture

Uses efficient I/O multiplexing with poll() to handle multiple clients simultaneously without blocking operations. Ensures high availability under heavy load.

Learn More

HTTP/1.1 Compliance

Implements the HTTP/1.1 specification with support for GET, POST, DELETE methods and accurate status codes. Compatible with all standard browsers.

Test Methods

CGI Execution

Execute Python, PHP, and Bash scripts through Common Gateway Interface with proper environment variables and directory handling.

Run Scripts

Configurable Server

Flexible configuration system with server blocks, ports, error pages, client body limits, and route-specific rules inspired by NGINX.

Configure Server

File Management

Supports file uploads through POST requests and directory listings. Clients can download or delete files through native HTTP methods.

Upload Files

Error Handling

Custom error pages with accurate HTTP status codes. Robust error recovery ensures the server remains operational under all conditions.
